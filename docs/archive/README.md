## When to Archive

- A doc is superseded by a newer canonical version, but the historical context is worth preserving (don't `git rm` it — let history be visible via the archive)
- A session artifact is older than 90 days and the founder hasn't referenced it (per [user-global CLAUDE.md §11 — When `sessions/` Accumulates](https://github.com/openova-io/openova/blob/main/CLAUDE.md))
- A proposal is rejected and we want to remember WHY it was rejected
- Research data that informed a decision (the data itself, not the decision — the decision lives in an ADR)

## When NOT to Archive

- One-off scratch files (delete those — `git log` is the history you need)
- Old README versions (those live in git history)
- Old PR descriptions (those live on GitHub)
